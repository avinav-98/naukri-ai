PROJECT: Naukri Auto Apply AI

STACK
FastAPI backend, Playwright automation, SQLite databases, Jinja templates frontend, async pipeline worker with polling UI. Local development at [http://127.0.0.1:8000](http://127.0.0.1:8000).

AUTH + UI
Routes: /signin, /signup, /dashboard, /jobs-directory, /fetch-jobs, /relevant-jobs, /applied-jobs, /settings, /profile.
Auth pages use auth layout; application modules use dashboard layout with sidebar. Resume upload occurs on /fetch-jobs page using .txt resume input.

PIPELINE FLOW
User uploads resume → links Naukri credentials → clicks Run Pipeline.
POST /api/fetch-jobs returns 202 Accepted with run_id.
Worker executes pipeline asynchronously:
fetch → rank → apply.
UI polls GET /api/pipeline-runs/{run_id} every 4s for status (queued → running → completed/failed).

AUTOMATION
Playwright browser automation logs into Naukri and scrapes jobs.
Thread-local Playwright instances are used to prevent greenlet/thread conflicts.
Login endpoint uses threadpool execution to avoid running Sync Playwright inside FastAPI asyncio loop.

JOB FETCH LOGIC
Search query dynamically determined:

1. Control Panel job_role (highest priority)
2. Otherwise inferred from resume text.

Resume parsing uses keyword scoring against ROLE_HINTS to infer role (data analyst, backend developer, devops engineer, etc.).
job_scraper builds dynamic URL:
[https://www.naukri.com/{slug}-jobs](https://www.naukri.com/{slug}-jobs)

DATA FLOW
Jobs scraped → stored in jobs_directory table with user_id.
Ranking engine scores jobs against resume.
Relevant jobs stored for review and auto-apply stage.

DATA HYGIENE
Before each new pipeline run the user's jobs_directory rows are cleared to prevent mixing results from previous queries.

DATABASE
SQLite with runtime migration system. db_migrations.py automatically adds missing columns (e.g., user_id) on startup to maintain backward compatibility.

SECURITY
JWT auth implemented with 32+ char secret key.
Session cookies used for authenticated routes.

CURRENT SYSTEM STATUS
Auth working.
Dashboard modules rendering correctly.
Async pipeline operational.
Playwright threading issues resolved.
Dynamic resume-driven job search implemented.
Control panel override working.

NEXT ARCHITECTURE UPGRADE
Replace in-process worker with Celery + Redis queue for distributed execution, crash resilience, and horizontal scaling.
